# 04. 새로운 인터페이스로 기존 루틴 부르기

- 기존 인터페이스 때문에 복잡하거나 어려울 경우 새로운 인터페이스를 구현해서 호출하라.
- 새로운 인터페이스에서 기존 인터페이스를 호출하는 것으로 구현 가능
- 새롭게 구현한 통로 인터페이스(pass-through interface)는 소프트웨어 설계에서 작은 단위로 중추적 역할
- 비슷한 느낌
  - 거꾸로 코딩하기 : 루틴의 마지막 줄부터 시작. 이미 상위에 로직은 작성 된것 처럼.
  - 테스트 우선 코딩 : TDD
  - 도우미 설계 : 특정 업무를 해 주는 루틴, 객체, 서비스가 있다면 나머지 작업은 쉬워진다.
